# Команды для работы с nvs
В корне json должен быть только один элемент __"nvs"__. Корень json может содержать другие элементы. 
Одновременно передаётся только одна команда, следующая только после получения ответа.

### 1.Очистить nvs.
```
{
    "nvs":
    {
        "clear":null
    }
}
```
Ответ
```
{
    "nvs":null
}
```
### 2.Прочитать данные.
```
{
    "nvs":
    {
        "name":"rssi",  //имя поля
        "type":"u16"    //тип поля (u8, u16, u32, i8, i16, i32, float, double, string, binary) по умолчанию u16
    }
}
```
Ответ
```
{
    "nvs":
    {
        "name":"rssi",  //имя поля
        "value":300     //значение поля, если поле не существует, то оно отсутствует
    }
}
```
### 3.Записать данные.
```
{
    "nvs":
    {
        "name":"rssi",  //имя поля
        "type":"u16",    //тип поля (u8, u16, u32, i8, i16, i32, float, double, string, binary) по умолчанию для целого числа i16, с точкой - float, иначе string
        "mode":1  //область записи: 1 - в основную область памяти (по умолчанию), 2 - в резервную (если возможно), 3 - в основную и резервную (если возможно)
        "value":300     //значение поля
    }
}
```
Ответ
```
{
    "nvs":
    {
        "name":"rssi",  //имя поля
        "value":300     //значение поля
    }
}
```
### 4.Запрет записи резервную область памяти.
```
{
    "nvs":
    {
        "lock":null  
    }
}
```
### 5.Перегрузить устройство.
```
{
    "nvs":
    {
        "reset":null  
    }
}
```
